# Processo de Desenvolvimento

## Escolha do Board: Kanban

O Kanban foi escolhido para o gerenciamento do desenvolvimento do ESM Forum por ser um modelo visual e contínuo de organização do trabalho.

Diferente do Scrum, o Kanban não exige iterações fixas (sprints), o que se adapta melhor a este projeto acadêmico com entregas incrementais e funcionalidades bem definidas.

Além disso, o Kanban permite:

- Visualização clara do fluxo de trabalho
- Controle de tarefas em andamento
- Priorização dinâmica das funcionalidades
- Facilidade de acompanhamento do progresso

Por esses motivos, ele é adequado para organizar as 5 funcionalidades solicitadas no projeto.

## Estrutura do Board

O board foi estruturado utilizando colunas no modelo Kanban:

- Backlog: funcionalidades planejadas e priorizadas
- In Progress: tarefas em desenvolvimento (não utilizado ainda)
- In Review: tarefas em revisão (não utilizado ainda)
- Done: tarefas concluídas

As funcionalidades foram adicionadas como issues e organizadas no Backlog seguindo a seguinte ordem de prioridade:

1. Busca de perguntas por palavra-chave
2. Sistema de votação em perguntas
3. Categorização de perguntas (tags)
4. Perfil de usuário com histórico
5. Notificação de novas respostas
